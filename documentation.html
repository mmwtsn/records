records

application documentation

Background

Records is a web app for helping to keep track of a small record collection. Last summer I picked up a used record player on Craigslist and discovered that vinyl and digital media compliment one another quite nicely as much of the music that is unavailable through streaming services (e.g. Rdio or Spotify) is released on vinyl. Given that, I was interested in building an app to track my nascent record collection.

Getting Started

To get started, open a new Terminal shell and either download the source code from CS50 Submit or, if you have Git installed, clone the repository from GitHub by executing: git clone https://github.com/mmwtsn/records.git. If you downloaded the archive, unzip it by executing unzip records.zip. Now that you have the source code, go ahead and execute cd records to navigate into the project directory.

System Dependencies

Records is built using Ruby on Rails atop the Discogs API. While Rails is a pleasure to use, it does come with a fair number of dependencies. Check to see if you have Ruby installed by executing which ruby. If no output is shown, you must first install Rubybefore proceeding. Once Ruby is installed, ensure that Ruby's package manager was installed correctly by executing which gem as you will need it to install the remainder of the development dependencies. Again, if no output is shown, install RubyGems before proceeding. Finally, check to see that your local instance of MySQL is installed properly and running by executing which mysql and mysqladmin status. MySQL should be installed already in most operating systems. If it's not installed on your computer, check the MySQL documentation for the latest installion instructions.

Development Dependencies

Once Ruby, RubyGems and MySQL are installed properly, we'll use Bundler to download and install the rest of the app's local dependencies. Install Bundler by running gem install bundler. From inside the records directory, Bundle the remaining libraries by running bundle install.

Local Setup

With all of the dependcies installed, you're ready to get started. Run rake db:setup && rake db:migrate to create the database, load the database schema and seed the database. Once that's complete, the migrate command will ensure that your database tables are set up properly. If interested, you can run the test suite by executing rake spec, though, this comes with added dependcies that you can learn more about in the capybara-webkit documentation. You are all set! Start a local server by running rails server and navigate your browswer of choice to http://localhost:3000 to use the app.

Finally, as the app actively consumes the Discogs API, you will need an internet connection to use the app.

Using Records

Using records is quite simple. From the home page, enter the name of an artist you are interested in (e.g. "jamie xx") and click the search button. Records will query the Discogs API and return a list of matches. Click any results you are interested in saving and it will automatically be saved to your collection and will appear below your search in your collection. From here, you can continue searching, clear your search with "clear" or scroll down to view your collection. If a search result is already in your collection, a green stripe will appear on the right of the result indicating that it has already been saved.

To view information about the tracklist of each record in your collection, click the view link next to the record once it has been saved. This will open a new view displaying the track list, location on the vinyl (side A1, etc.) and trak duration. Records can be removed from your collection by clicking the delete link.
